1. 클래스(Class)
- 객체를 만드는 설계도
- new 키워드를 사용해 객체를 생성할 수 있다.

2. 객체(Object)
- 클래스를 바탕으로 만들어진 실체
- 클래스에 정의된 변수들과 함수를 갖고 있는 독립적인 대상

3. MonoBehaviour
- 유니티에서 제공하는 특별한 클래스
- new 키워드를 직접 사용하지 않고  AddComponent() 함수로 게임오브젝트에 
컴포넌트 객체를 만들어 사용한다.
- 인스펙터뷰에서 참조 변수 연결이 가능하다.

4. 일반 객체(비 MonoBehaviour 객체)
- 우리가 직접 만드는 C# 클래스에서 new 키워드를 통해 만든 객체
- MonoBehaviour의 기능들은 없지만, 주로 상태 관리, 데이터 묶음 등에 유용하게 사용한다.
- 인스펙터뷰에서 참조 변수 연결이 불가능하다.
- 코드상에서 new 키워드로 객체를 만들어 사용한다.
- [System.Serializable]을 사용해 인스펙터뷰에서 편집 가능하게 할 수 있다.

5. 스크립터블 오브젝트(Scriptable Object)
- 유니티에서 제공하는 특별한 클래스
- 컴포넌트로 사용되지 않는 일종의 에셋 파일
- 인스펙터뷰에서 편집 가능하다.

=====
객체 지향의 4가지 핵심 요소
1. 캡슐화(Encapsulation)

2. 상속(Inheritance)

3. 다형성(Polymorphism)

4. 추상화(Abstraction)
